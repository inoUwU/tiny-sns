# tiny-sns DBML不足項目洗い出し

## 1. 目的

この文書は、元の `docs/sns.dbml` に対して MVP の観点で不足していた項目を整理し、今回どのようにスキーマへ反映したかを説明するための補助資料です。

## 2. 不足項目サマリ

| 不足項目 | 影響 | 今回の反映 |
| --- | --- | --- |
| 非公開アカウント設定がない | `follows.status=pending` の意味が確定しない | `users.is_private` を追加 |
| ログインセッションの保存先がない | ログアウト、複数端末、失効管理ができない | `auth_sessions` を追加 |
| メール確認状態がない | 新規登録後の本人確認を表現できない | `users.email_verified_at` と `email_verification_tokens` を追加 |
| パスワード再設定の保存先がない | パスワード紛失時の復旧導線が持てない | `password_reset_tokens` を追加 |
| 通報を保存できない | 安全機能がブロック/ミュート止まりになる | `reports` を追加 |
| タイムライン系インデックスが不足 | MVP でもホーム表示が重くなりやすい | `posts`、`follows`、`notifications` に索引を追加 |
| タグ/メンションの逆引き索引が不足 | タグ別投稿一覧や通知生成が非効率 | `post_hashtags`、`mentions` に索引を追加 |

## 4. DBMLへ入れなかった項目

以下は必要性が高い一方で、RDB のコアスキーマよりもアプリケーション層またはインフラ層で表現する方が自然なため、今回は DBML に含めていません。

- 画像・動画の保存バケット、変換ジョブ、CDN 配信設定
- Redis ベースのタイムラインキャッシュ、ジョブキュー、ワーカー状態
- 専用検索エンジン、ランキング、トレンド集計
- API レート制限、Bot 判定、監査ログ

## 5. 移行時の注意

- 既存ユーザーは `is_private=false` を初期値としてバックフィルする
- `email_verified_at` は既存ユーザーでは `null` 開始とし、検証ポリシーを別途定義する
- `auth_sessions` とトークンテーブルは平文ではなくハッシュ化済みトークンを保存する
- `reports` は `target_user_id` または `target_post_id` のどちらか一方を必須とする制約をアプリケーション側で担保する
